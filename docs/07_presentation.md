# CarbonLand 5분 발표 시나리오

> 작성일: 2025-12-13
> 대상: 경기 기후 바이브코딩 해커톤 2025

## 발표 개요
- **시간**: 5분 (300초)
- **형식**: 라이브 데모 + 설명
- **준비물**: 브라우저에 CarbonLand 메인 페이지 열어둠

---

## 시나리오 타임라인

### 0:00-0:30 | 인트로 (30초)
**화면**: 메인 페이지 (지도 화면)

**멘트**:
> "안녕하세요, CarbonLand를 소개합니다.
>
> **'이 땅을 개발하면, 탄소는 얼마나 변할까?'**
>
> 우리 주변의 녹지가 아파트가 되고, 산림이 공장이 될 때
> 탄소는 얼마나 줄어들까요?
>
> CarbonLand는 경기기후플랫폼의 실제 데이터를 활용해서
> 토지이용 변경에 따른 탄소 영향을 실시간으로 시뮬레이션하는 도구입니다."

---

### 0:30-1:00 | 문제 정의 (30초)
**화면**: 그대로 메인 페이지

**멘트**:
> "현재 개발사업 기획 시 탄소 영향을 파악하기 어렵습니다.
>
> 환경영향평가는 비용과 시간이 많이 들고,
> 시민들은 개발의 환경적 영향을 이해하기 어렵습니다.
>
> CarbonLand는 이 문제를 해결합니다.
> 누구나, 어디서든, 즉시 탄소 영향을 확인할 수 있습니다."

---

### 1:00-1:40 | 기능 1: 주소 검색 (40초)
**화면**: 컨트롤 패널 - 주소 검색

**액션**:
1. 주소 검색창에 **"광교산"** 입력
2. 검색 버튼 클릭
3. **"광교산"** 결과 선택 → 지도 이동

**멘트**:
> "먼저 분석하고 싶은 지역을 찾습니다.
>
> **'광교산'**을 검색해볼게요.
> 이 산림 지역에 개발이 계획된다고 가정하겠습니다.
>
> (결과 선택 후) 광교산 주변으로 지도가 이동됩니다."

---

### 1:40-2:20 | 기능 2: 직접 영역 그리기 + 자동감지 (40초)
**화면**: 컨트롤 패널 → 영역 그리기 버튼 + 오른쪽 분석 결과 패널

**액션**:
1. "영역 그리기" 버튼 클릭
2. **지도 중앙의 녹색 산림 지역**에 사각형으로 그리기
3. 4번 클릭 후 더블클릭으로 완료 → **"자동감지"** 배지 확인
4. 오른쪽 **분석 결과 패널** 확인 (시나리오별 탄소 현황 비교)

**멘트**:
> "이제 분석할 영역을 직접 그려보겠습니다.
>
> '영역 그리기' 버튼을 클릭하고...
> **광교산의 녹색 산림 지역**을 사각형으로 선택합니다.
>
> (4번 클릭 후 더블클릭으로 완료)
>
> 보시면 **'자동감지'** 배지가 뜨죠?
> 경기기후플랫폼 WFS API가 이 영역을 분석해서
> **'산림'**으로 자동 감지했습니다.
>
> 오른쪽 **분석 결과 패널**을 보시면,
> 영역을 선택하자마자 **7개 시나리오별 탄소 현황**이 바로 비교됩니다.
> 현재 산림 상태를 유지할 때, 주거지/상업지/공업지로 개발할 때,
> 습지나 공원녹지로 조성할 때의 탄소 영향을 **즉시** 확인할 수 있습니다."

---

### 2:20-2:50 | 기능 3: 시뮬레이션 실행 (30초)
**화면**: 컨트롤 패널 → 시뮬레이션 설정

**액션**:
1. 변경 시나리오: **"주거지 개발"** 선택
2. 분석 기간: **30년** 설정
3. **"시뮬레이션 실행"** 버튼 클릭

**멘트**:
> "이 산림 지역에 **아파트 단지**를 개발한다고 가정해볼게요.
>
> 변경 시나리오에서 **'주거지 개발'**을 선택하고,
> 분석 기간은 **30년**으로 설정합니다.
>
> (버튼 클릭) 시뮬레이션을 실행하면..."

---

### 2:50-3:40 | 기능 4: 결과 분석 (50초)
**화면**: 결과 패널 (모바일이면 결과 탭 클릭)

**액션**:
1. 결과 패널 스크롤하며 각 섹션 설명
2. 환산 지표 강조
3. 시나리오 비교 테이블 보여주기

**멘트**:
> "결과가 나왔습니다.
>
> **즉시 배출량** - 산림을 제거할 때 방출되는 탄소입니다.
>
> **순 탄소수지가 마이너스**로 바뀌었죠?
> 탄소를 흡수하던 땅이 배출하는 땅으로 바뀐 겁니다.
>
> 이게 어느 정도냐면...
> **30년생 소나무 수천 그루**가 흡수하는 양이고,
> **승용차 수천 대**의 연간 배출량입니다.
>
> 아래 **전체 시나리오 비교**를 보시면,
> 7개 시나리오를 한눈에 비교할 수 있습니다.
> 별표가 붙은 게 탄소중립 관점에서 최적 시나리오입니다."

---

### 3:40-4:05 | 기능 5: WMS 레이어 (25초)
**화면**: 레이어 패널

**액션**:
1. 레이어 탭/패널 열기
2. "탄소흡수지도" 레이어 켜기
3. 지도에 레이어 표시 확인

**멘트**:
> "경기기후플랫폼의 실제 데이터를 지도에서 직접 볼 수도 있습니다.
>
> 탄소흡수지도 레이어를 켜면...
> 녹색이 진할수록 탄소 흡수량이 높은 지역입니다."

---

### 4:05-4:30 | 기능 6: 결과 저장/공유 (25초)
**화면**: 결과 패널 - 저장/공유 버튼

**액션**:
1. "저장/공유" 버튼 클릭
2. 드롭다운 메뉴 보여주기
3. "PDF 리포트 저장" 또는 "이미지 저장" 클릭

**멘트**:
> "분석 결과는 저장하고 공유할 수 있습니다.
>
> 이미지, PDF 리포트, URL 공유까지...
> 회의 자료나 보고서에 바로 활용할 수 있습니다."

---

### 4:30-5:00 | 마무리 (30초)
**화면**: 프로젝트 소개 페이지 (/about)

**액션**:
1. 상단 "프로젝트 소개" 클릭
2. 핵심 특징 배너 보여주기

**멘트**:
> "정리하면, CarbonLand는
>
> 1. **경기기후플랫폼 API**를 실제로 연동해서
> 2. **비오톱을 자동 감지**하고
> 3. **7개 시나리오를 비교**할 수 있으며
> 4. **결과를 저장/공유**할 수 있습니다.
>
> 공무원, 개발사업자, 시민 누구나
> 개발의 탄소 영향을 쉽게 이해할 수 있는 도구입니다.
>
> 감사합니다."

## 발표 체크리스트

### 발표 전 준비
- [ ] 브라우저에 CarbonLand 메인 페이지 열기
- [ ] 인터넷 연결 확인 (API 호출용)
- [ ] 화면 확대/축소 적절히 조정
- [ ] 불필요한 탭/알림 끄기

### 데모 시 주의사항
- 클릭 전에 잠시 멈추고 "~을 클릭하면" 말하기
- 결과 로딩 시간 활용해서 설명 추가
- API 에러 시 "시뮬레이션 데이터로 대체됩니다" 언급

### 예상 질문 대비
1. **데이터 정확도**: "IPCC 가이드라인 기반, 실제 환경영향평가와 동일한 계수 사용"
2. **API 출처**: "경기기후플랫폼 Open & Share에서 제공하는 공식 API"
3. **향후 계획**: "건물 배출량 연계, 시계열 분석 고도화 예정"
